Miniconda Installation & Environment Setup

A comprehensive guide for installing miniconda and environment setup

üìå Step 1: Open a Terminal

Open your preferred terminal application (e.g., GNOME Terminal, Konsole, xterm).

üìå Step 2: Download the Installer

Run the command to download the latest Miniconda installer for 64-bit Linux:

wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
üîç Tip: If you're using an ARM-based system (e.g., Raspberry Pi, Apple M1/M2 via VM), you'll need the aarch64 installer from the official Miniconda website.

üìå Step 3: Verify the Installer (Optional but Recommended)

Check the installer's integrity against its SHA256 hash.

sha256sum Miniconda3-latest-Linux-x86_64.sh

Compare the output with the hash listed on the official Miniconda website to ensure they match.

üìå Step 4: Make the Installer Executable 

chmod +x Miniconda3-latest-Linux-x86_64.sh

üìå Step 5: Run the Installer

Start the interactive setup:

./Miniconda3-latest-Linux-x86_64.sh

Then follow the on-screen prompts:

  • Press Enter to scroll through the license agreement.
  • Type yes and press Enter to accept the license.
  • Press Enter to confirm the installation directory (default is ~/miniconda3).
  • Type yes and press Enter to initialize Miniconda.

üìå Step 6: Reload the Shell

For the changes to take effect, close and reopen your terminal, or run:

source ~/.bashrc   # or source ~/.zshrc if you use Zsh

üìå Step 7: Test the Installation

Verify that Conda was installed correctly:

conda --version

Example output:

conda 24.3.0

üìå Step 8: Create and Manage an Environment

Create a new environment, activate it, and install packages.

# Create a new environment named 'myenv' with Python 3.11
conda create -n myenv python=3.10 -y

# Activate the environment
conda activate myenv

#Install from a requirements.txt file
conda install --file requirements.txt 
                or
pip install -r requirements.txt 

# Deactivate to return to the base environment
conda deactivate
